# Comugest Frontend — Pages Reference

All pages live in `src/pages/`. Each page is a named export (`export function XxxPage()`).

---

## Public Pages (no auth required)

### `/login` — LoginPage

**File**: `src/pages/Login.jsx`  
**Access**: Public  
**Description**: Email + password form. Handles the 2FA challenge flow — if the API returns `requiresTwoFactor: true`, renders a TOTP code input.  
**API calls**: `POST /auth/login`, `POST /auth/login/2fa`  
**Redirects to**: `/` on success

---

### `/register` — RegisterPage

**File**: `src/pages/Register.jsx`  
**Access**: Public  
**Description**: Account creation form (firstName, lastName, email, password, phone). On success, redirects to `/check-email`.  
**API calls**: `POST /auth/register`

---

### `/soporte` — SoporteLoginPage

**File**: `src/pages/SoporteLogin.jsx`  
**Access**: Public  
**Description**: Separate login page for Comugest Support staff. Functionally the same as `/login` but branded differently and intended for internal use.  
**API calls**: `POST /auth/login`

---

### `/check-email` — CheckEmailPage

**File**: `src/pages/CheckEmail.jsx`  
**Access**: Public  
**Description**: Static confirmation screen shown after registration. Prompts the user to check their inbox to verify their email.  
**API calls**: None (static)

---

### `/verify-email` — VerifyEmailPage

**File**: `src/pages/VerifyEmail.jsx`  
**Access**: Public  
**Description**: Reads `?token=` from the URL and calls the verify-email endpoint. Shows success or error state.  
**API calls**: `POST /auth/verify-email`

---

### `/accept-invitation` — AcceptInvitationPage

**File**: `src/pages/AcceptInvitation.jsx`  
**Access**: Public  
**Description**: Accepts a resident invitation. Reads `?token=` from URL. Renders a form to set a password and complete the onboarding.  
**API calls**: `POST /invitations/accept`

---

### `/forgot-password` — ForgotPasswordPage

**File**: `src/pages/ForgotPassword.jsx`  
**Access**: Public  
**Description**: Email input form that sends a password reset link.  
**API calls**: `POST /auth/forgot-password`

---

### `/reset-password` — ResetPasswordPage

**File**: `src/pages/ResetPassword.jsx`  
**Access**: Public  
**Description**: New password form. Reads `?token=` from URL.  
**API calls**: `POST /auth/reset-password`

---

### `/meetings/qr-check-in/:token` — QrCheckInPage

**File**: `src/pages/QrCheckIn.jsx`  
**Access**: Public (token in URL)  
**Description**: Handles QR code check-in for board meetings. The token is scanned from a QR code generated by the admin. Confirms attendance without requiring the resident to be logged in.  
**API calls**: `POST /meetings/qr-check-in/:token`

---

## Resident Pages (any authenticated user)

### `/` — DashboardPage

**File**: `src/pages/Dashboard.jsx`  
**Access**: Any authenticated user  
**Description**: Central dashboard. For admins: shows community KPIs, recent activity, pending actions. For residents: shows pending invoices, upcoming meetings, recent announcements, procedure status.  
**Key components**: `Layout`, `OnboardingWizard` (first login)  
**API calls**: `GET /admin/kpis` (admin), `GET /me/communities`, `GET /me/invoice-items`, `GET /me/meetings`

---

### `/my-invoices` — MyInvoicesPage

**File**: `src/pages/MyInvoices.jsx`  
**Access**: Any authenticated user  
**Description**: Lists all invoice items assigned to the resident. Shows amount, due date, payment status, community name.

## Admin Pages (ADMIN_FINCAS + SUPPORT)

### `/communities` — CommunitiesListPage

---

### `/communities/new` — CreateCommunityPage

**Description**: Form to create a new community (name, address, city, postal code, CIF).  
**API calls**: `POST /communities`

---

### `/communities/:id` — CommunityDetailPage

**File**: `src/pages/CommunityDetail.jsx`  
**Access**: ADMIN_FINCAS, SUPPORT  
**Description**: Community overview hub. Shows community info, units list with owner/occupant details, co-admins, onboarding status. Links to all sub-sections. Allows editing community metadata and managing custom field definitions.

**API calls**: `GET /communities/:id`, `GET /communities/:id/units`, `PATCH /communities/:id`, `GET /communities/:id/co-admins`

### `/communities/:id/invoices` — CommunityInvoicesPage

**File**: `src/pages/CommunityInvoices.jsx`  
**Access**: ADMIN_FINCAS, SUPPORT  
**Description**: Paginated table of all invoices for the community. Filter by date, type, status. Shows outstanding totals. Links to invoice detail and overdue list.  
**API calls**: `GET /communities/:id/invoices`, `GET /communities/:id/invoices/overdue`

---

### `/communities/:id/invoices/new` — CreateInvoicePage

**File**: `src/pages/CreateInvoice.jsx`  
**Access**: ADMIN_FINCAS, SUPPORT  
**Description**: Two-mode invoice creation: DERRAMA (total amount distributed by coefficient) or INDIVIDUAL (per-unit amounts with optional consumption data). Preview of generated items before saving.

### `/invoices/:id` — InvoiceDetailPage
